Marven Market Dashboard

Project Brief

The Maven Market dataset analyzed a comprehensive insights into the overall performance of Maven Market, a multi-national grocery chain with locations in Canada, Mexico, and the United States. It produced useful insights by identifying new markets, goods, and advertising opportunities and by utilizing forecasts to comprehend upcoming trends and consumer behavior.

Project Goals

Developing a Power BI dashboard to analyze transactions and sales and provide information on pick time distribution and monthly and annual trends.

Data Source

The dataset consists of raw Excel files provided by Maven Analytics where includes data about customers, products, stores, transactions & returns;

View dataset


Business Request & User Stories

As a participant in the Microsoft PowerBI Desktop for Business Intelligence (2023) course by Maven Analytics, I have developed the Maven Market Report. The report is designed to enhance the learning experience of course participants by combining the course instructions with creative elements to showcase practical application and analytical skills.

Data Cleansing and ETL

Some insight of the project workflow were done in Power BI for analysis purposes:

1. Corrected data types, promoted headers, and cleaned null values and duplicates.
2. Added new columns for date-time analysis, such as the start of the week and the start of the year.
3. Create a new column at table Customer, named as "birth_year"to extract the year from the "birthdate" column, and format as text
4. Create a conditional column at table Customer, named as "has_children" which equals "N" if "total_children"= 0, otherwise "Y"
5. Replace "null" values with zeros in related columns.
6. Add a calculated column at table Products, named as "discount_price", equal to 90% of the original retail price. Format as a fixed decimal number, and then use the rounding tool to round to 2 digits.

Calculations

The following calculations were created in the Power BI reports using DAX (Data Analysis Expressions):

60-Day Revenue =
CALCULATE([Total Revenue],
DATESINPERIOD('Calendar'[date],
MAX('Calendar'[date]),-60,DAY))

% Weekend Transactions = [Weekend Transactions] / [Total Transactions]

All Returns = CALCULATE([Total Returns], ALL(Returns))

All Transactions = CALCULATE([Total Transactions], ALL(Transaction_Data))

Last Month Profit =
CALCULATE([Total Profit],
DATEADD('Calendar'[date],-1,MONTH))

Last Month Returns =
CALCULATE([Total Returns],
DATEADD('Calendar'[date],-1,MONTH))

Last Month Revenue =
CALCULATE([Total Revenue],
DATEADD('Calendar'[date],-1,MONTH))

Last Month Transactions =
CALCULATE([Total Transactions],
DATEADD('Calendar'[date],-1,MONTH))

Profit Margin = [Total Profit] / [Total Revenue]

Quantity Returns = SUM(Returns[quantity])

Quantity Sold = SUM(Transaction_Data[quantity])

Return Rate = [Quantity Returns] / [Quantity Sold]

Revenue Target = [Last Month Revenue] * 1.05

Total Cost = SUMX(Transaction_Data, Transaction_Data[quantity] * RELATED(Products[product_cost]))

Total Revenue = SUMX(Transaction_Data, Transaction_Data[quantity] * RELATED(Products[product_retail_price]))

Total Profit = Total Revenue] - [Total Cost]

Total Returns = COUNTROWS(Returns)

Total Transactions = COUNTROWS(Transaction_Data)

Unique Products = DISTINCTCOUNT(Products[product_name])

Weekend Transactions = CALCULATE([Total Transactions], 'Calendar'[Weekend] = "Y")

YTD Revenue = CALCULATE([Total Revenue], DATESYTD('Calendar'[date]))
